## Project Summary and Problem It Was Solving

The projects I selected from this course were the Thermostat final project and Milestone 3. These projects focused on connecting software with hardware components using a Raspberry Pi. The Thermostat project solved the problem of monitoring room temperature and allowing the user to control heating and cooling modes with buttons and LED indicators. Milestone 3 focused on using button input and a state machine to control outputs, which helped demonstrate how embedded systems react to user interaction in real time.

These projects reflect the purpose of emerging systems architectures because they combine programming, sensors, user input, and hardware control into one working system.

## What I Did Particularly Well

I believe I did particularly well in troubleshooting and staying persistent when problems appeared. Many times, hardware projects do not work perfectly on the first try, so I had to carefully test wiring, button inputs, GPIO pins, and software logic. I also did well organizing the logic of the thermostat system by separating modes such as OFF, HEAT, and COOL.

Another strength was learning how software decisions directly affect hardware behavior. For example, using LED fading and button controls made the system more interactive and realistic.

## Where I Could Improve

One area where I could improve is spending even more time planning before starting to wire or code. Sometimes I learned through trial and error, which was valuable, but stronger planning could save time. I would also like to improve my confidence with electronics concepts such as resistors, pull-up and pull-down logic, and circuit design.

In future projects, I would also like to make the user interface more advanced, such as using a display screen or remote access dashboard.

## Maintainable, Readable, and Adaptable Design

I made these projects maintainable and readable by organizing the code into clear sections, using logical variable names, and separating functions when possible. I also used comments to explain important parts of the program.

The projects were adaptable because settings such as temperature targets, GPIO pins, timing values, and system states could be changed without rewriting the full program. This makes future improvements easier and allows the same structure to be reused in other embedded system projects.
